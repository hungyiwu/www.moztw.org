Firefox 新鮮事

  • Firefox 78，此版本支援 10.9、10.10 與 10.11 版的 macOS。如果您是這三種 macOS 版本的用戶，接下來一年都將享有 Firefox 78.x 長期支援版 (Extended Support Release，ESR) 的穩定支援。
  • 全新功能 保護資訊儀錶板彙整關於追蹤保護、資料外洩事件與密碼管理的摘要報告。新功能可讓使用者：
    • 在儀錶板中檢閱已處理的資料外洩事件數目
    • 檢查已儲存的密碼是否曾於外洩事件中洩漏
    若要進入儀錶板，您可在 Firefox 網址列輸入 about:protections，或在主選單中按「保護資訊儀錶板」。
  • 全新功能 因為我們知道，有些人會為了解決問題而重新安裝 Firefox，但其實重新整理可能是更有效的做法，所以，我們幫 Firefox 的卸載程式增加｢重新整理」(Refresh) 的按鍵。
  • 全新功能 螢幕保護程式將無法再干擾此版本 Firefox 上的 WebRTC 呼叫，而能帶來更流暢的視訊與會議體驗。
  • 全新功能 針對內建 內建 Intel GPU 的 Windows 電腦用戶釋出 WebRender，讓更多使用者都能享受到更高的繪圖效能。
  • 全新功能 Firefox 78 也是長期支援版 (ESR) 的產品，也就是說，我們將對 ESR 使用者同步釋出 Firefox 過去 10 次更新所做的變更。值得一提的功能與特點包括：
    • Kiosk 模式
    • 用戶端憑證
    • 支援 Service Worker 與 Push API
    • 增加阻擋自動播放的功能
    • 支援子母畫面 (picture-in-picture)
    • 在 about:certificate 中檢視與管理網頁憑證
  • 全新功能 Pocket 推薦精選網路上的奇文妙作，英國境內所有的 Firefox 使用者都可在新分頁中看到這些精彩文章。如果您人在英國，但沒看到這些內容的話，您可到新分頁打開 Pocket 文章。請到此參閱步驟說明。
  • 修正 修補幾種不同的安全弱點。
  • 修正 修復影響搜尋結果品質的程式缺陷，並依合作夥伴的建議改善搜尋結果的文字。
  • 變更 Linux 的最低系統需求已有所調整，Firefox 現在需要 GNU libc 2.17、libstdc++ 4.8.1 和 GTK+ 3.14 或以上版本。
  • 變更 配合我們淘汰過時密碼學的策略，僅剩的一些基於 DHE 的 TLS 密碼套件已全都預設關閉。
    • 為解決因停用此種密碼套件而產生的網頁相容性問題，Firefox 78 另採用兩個基於 AES-GCM SHA2 的密碼套件。
  • 變更 停用 TLS 1.0 與 TLS 1.1 ，以改善網站連線品質。不支援 TLS 1.2 的網站將顯示錯誤頁。
  • 變更 在右鍵選單 (在分頁上點滑鼠右鍵即可叫出) 中，只按一鍵便能復原多個已關閉的分頁，並可於子選單上點選「關閉右方分頁」或「關閉其他分頁」。
  • 變更 優化了幾個無障礙體驗：
    • 當使用 JAWS 螢幕閱讀器時，點選具有資料清單的 HTML 輸入控制項中的下箭頭不再錯誤地移動游標到輸入控制項的下一個元素。
    • 在螢幕閱讀器上聚焦到麥克風、相機與螢幕分享指示器時，不再導致嚴重的 Lag 或停止回應。
    • 大幅提高了有數千行的表格對於螢幕閱讀器的使用者的載入速度。
    • 文字輸入控制項在有自訂樣式時，能夠適當的顯示聚焦輪廓。
    • 在螢幕閱讀器開啟主要的開發者工具視窗時，不再會錯誤的切換成文件瀏覽模式。
    • 減少像是懸停分頁、展開搜尋列之類的動畫，以降低對患有頭痛與癲癇使用者的視覺刺激。
  • 企業版 將 security.osclientcerts.autoload 的偏好設定設成 true，即可啟用對儲存在 macOS 和 Windows 系統上用戶端憑證的實驗性支援。
  • 企業版 新政策可讓您配置應用程式的處理程序、停用子母畫面、要求使用者設定主控密碼 (未來將改名為「主密碼 (primary password)」)。
  • 企業版 詳細資訊請參閱企業版 Firefox 78 版本資訊。
  • 開發功能 DevTools 主控台現在會更詳細記錄未捕捉的 promise 錯誤，提供更多關於名稱 (names)、堆疊 (stacks) 與屬性 (properties) 的訊息，而特別有助於加強 JavaScript 框架除錯的效率。
  • 開發功能 除錯器 (Debugger) 的最小變數名稱自動映射功能現亦支援 LogPoints 可提升源映射 (source-mapped) 專案除錯的流暢度。
  • 開發功能 Firefox DevTools’ Network 面板現在會顯著顯示哪一個擴充套件或 CORS (跨來源資源共用) 的限制造成請求遭到封鎖，以便開發者改善網站的復原性與安全性。
  • 開發功能 SpiderMonkey 新的 RegExp 引擎功能已進一步強化，增加對 dotAll flag、Unicode 脫逸序列 (escape sequences)、lookbehind 參照和命名捕捉 (named captures) 的支援。
  • 請參考此版本的完整變更清單。 您可能也想瞭解前一個版本的更新。
